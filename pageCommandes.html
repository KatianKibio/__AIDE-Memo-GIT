# Commandes Actions Utilisations
INTERFACE GRAPHIQUE
. gitk Ouvre une interface graphique du dépôt local gitk
CONFIGURATION / INITIALISATION
. git --version Affiche la version actuel
. git config configuration git config --global user.name "John Doe"
git config --global user.email sam@google.com
. git init Initialiser un nouveau projet git init (dans le dossier à initialiser)
ACTION EN DISTANT
. git clone La commande git clone est utilisée pour la vérification des dépôts. Si le dépôt se trouve sur un serveur distant, utilisez:

Inversement, si une copie de travail d’un dépôt local doit être créée, utilisez: 		git clone [url]
git clone [url] [nouveau_nom]

git clone /chemin/vers/dépôt
. git fetch permet à un utilisateur d’extraire tous les fichiers du dépôt distant qui ne sont pas actuellement dans le répertoire de travail local :
mais ne modifie pas le depot local		 git fetch origin
. git merge La commande git merge va quant à elle intégrer localement les modifications présentes sur le dépôt distant. :
git merge origin
. git pull git fatch >> git merge. fusionner toutes les modifications présentes sur le dépôt distant dans le répertoire de travail local : git pull
. git push Cette commande prend en paramètres le nom du dépôt distant suivi du nom de la branche cible à mettre à jour.
Un simple push envoie les modifications locales apportées à la branche principale associée
La commande git push envoie les modifications locales vers le dépôt distant nommé origin et met à jour la branche exemple_pull_branch :		 git push origin master
. git remote add synchroniser le projet versionné localement avec son dépôt distant:
Celle-ci attend en paramètre le nom du dépôt distant. Par convention, le dépôt par défaut est nommé origin.
Il faut également préciser l'URL d'accès à ce dépôt. 		git remote add origin git@github.com:JohnDoeDemo/repository-exemple.git
. git remote -v répertorie les dépôts distants actuellement configurés: git remote -v
. git remote show répertorie les dépôts distants actuellement configurés: git remote show nom_du_depot
ACTION EN LOCAL
. git branch supprimer / créer / afficher git branch –d nom-branche
git branch newBranch
git branch
. git merge fusionner une branche dans la branche active : git merge (nom-branche)
. git checkout Crééer / Basculer créer :
command git checkout -b (nom-branche)

Basculer :
git checkout (nom-branche)
AFFICHAGE / RECHERCHE
. git show afficher des informations sur tout fichier git : git show
. git cat-file À l’aide de la valeur SHA-1, affichez le type d’un fichier : git cat-file –p d670460b4b4aece5915caf5c68d12f560a9fe3e4
. git grep rechercher dans les arbres de contenu des expressions et / ou des mots : git grep "www.hostinger.com"
ACTION D'ETATS
tracked [unmodified / modified / staged] - untracked
. git status affiche la liste des fichiers modifiés ainsi que les fichiers qui doivent encore être ajoutés ou validés : git status
. git add ajouter des fichiers à l’index : git add temp.txt
. git restore Anuller une modif (avant commit) : git restore fichier.html
. git restore --staged retire un fichier indexé (avant commit) : git restore --staged fichier.html
. git commit valider les modifications apportées au HEAD : git commit –m “Description du commit”
. git stash aide à enregistrer les changements qui ne doivent pas être commit immédiatement. C’est un commit temporaire : git stash
. git reset réinitialiser l’index et le répertoire de travail à l’état du dernier commit : git reset --hard HEAD
. git mv Déplace un dossier dans un dossier : git mv source destination
. git rm supprimer des fichiers de l’index et du répertoire de travail : git rm nomfichier.txt
. git rm --cached supprimer de l'index et garde rdans le répertoire de travail : git rm --cached nomfichier.txt
LOG / SHA
. git log génère le log d’une branche : git log
. git ls-tree afficher un fichier arborescent avec le nom et le mode de chaque élément, et la valeur SHA-1 du blob : git ls-tree HEAD
OPTIMISER
. git gc Pour optimiser le dépôt en supprimant les fichiers inutiles et les optimiser : git gc
SECURITE
. git fsck Pour effectuer une vérification d’intégrité du système de fichiers git,
Tous les fichiers corrompus seront identifiés :		 git fsck
. git diff 1- lister les conflits : 2- afficher les conflits entre les branches à fusionner avant de les fusionner 3- énumérer tous les conflits actuels : 1- git diff --base nom-fichier
2- git diff branche-source branche-cible
3- git diff
. git diff --color-words Le détail des éléments modifié se fera alors par mots, plutôt que par ligne. git diff --color-words
